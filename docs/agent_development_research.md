# Agent Development Research: Medication Intake Clarification

## Research Question

How should MedCombo develop the first version of its medication intake agent?

## Recommendation

Build the first agent as a deterministic slot-filling state machine, not as a
free-form autonomous LLM agent.

The agent should:

- Start from a draft medication intake list.
- Ask one factual clarification question at a time.
- Capture user answers into structured fields.
- Keep unresolved or uncertain answers visible.
- Recompute remaining questions after every answer.
- Produce review-packet-ready data for a pharmacist or clinician.

The first implementation should avoid direct medical decisions, medication
commands, and LLM-generated safety signals.

## Evidence Summary

| Evidence | Design Implication | Confidence |
| --- | --- | --- |
| AHRQ medication interview guidance recommends asking about prescriptions, OTC drugs, vitamins, herbals, supplements, inhalers, full dosing details, and probing questions that trigger patient memory. | The agent should ask targeted factual questions about identity, source, strength, dose, frequency, formulation, route, and last dose. | high |
| AHRQ MATCH frames medication reconciliation as a process of obtaining, verifying, documenting, and comparing medication information. | The agent should update structured intake state, not simply chat or summarize. | high |
| The AMREC 2025 medRxiv preprint describes a conversational medication reconciliation agent with explicit dialogue flows for confirming existing medications and collecting additional medications. It also reports failures around natural dialogue patterns and digressions. | MedCombo should define the flow and state transitions first, then add LLM parsing later. | medium |
| FDA transparency principles emphasize human-AI team performance, clear essential information, intended inputs and outputs, known limitations, workflow placement, and human-centered design. | The agent should expose what it knows, what it does not know, and why it is asking each question. | high |
| FDA CDS guidance says digital health policies continue to apply to device software functions, including patient or caregiver functions that meet the device definition. | The agent should remain a review-preparation tool and avoid treatment decisions. | high |
| CONSORT-AI emphasizes reporting intervention instructions, input and output handling, human-AI interaction, and error cases for AI interventions. | The development agent should log turns, captured fields, unresolved answers, and failure modes. | medium |
| NIST AI RMF promotes risk-aware design, measurement, governance, and management of AI systems. | The agent should be testable with scenario fixtures before any stronger product claims. | medium |

## Development-Level Agent Design

The first agent should have these objects:

- `MedicationAgentSession`: current intake items, active questions, answered
  question IDs, and turn history.
- `AgentTurn`: one answered question, extracted field, extracted value, and
  status.
- `ConversationQuestion`: already generated by the intake layer.
- `MedicationIntakeItem`: already tracks raw text, source confidence,
  normalized candidate, missing fields, and review questions.

The first state transitions should support:

- Unknown product answer -> attempt normalization from the answer.
- Ambiguous product answer -> select a candidate when the answer matches one.
- Information source answer -> set source confidence.
- Strength answer -> capture strength.
- Dose answer -> capture dose.
- Frequency answer -> capture frequency.
- Formulation answer -> capture formulation.
- Unknown or unsure answer -> keep the item unresolved and record the turn.

## Safety Boundary

The agent may ask:

- "Which product matches the label?"
- "What strength is listed?"
- "How often do you take it?"
- "Did this come from a label, pharmacy list, photo, or memory?"

The agent should not ask or say:

- "Should you take this?"
- "Do you want to stop this?"
- "This combination is safe."
- "This medication is unsafe."
- "You should change how you take this."

## Implementation Plan

1. Add session and turn models.
2. Add a deterministic `medcombo.agent` module.
3. Start a session from raw medication lines.
4. Accept one answer and update the corresponding intake item.
5. Regenerate active questions while skipping answered questions.
6. Add tests for identity selection, source capture, field capture, and unsure
   answers.
